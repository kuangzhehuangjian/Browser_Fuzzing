code
directive in module code

Description

Using Angular markup like code in a code attribute doesn't work right: The browser will fetch from the URL with the literal text code until Angular replaces the expression inside code. The code directive solves this problem.

The buggy way to write it: 

pre

The correct way to write it: 

pre

Usage

as attribute
pre

Directive info

  • This directive executes at priority level 99.

Parameters

ParamTypeDetails
ngSrctemplate

any string which can contain code markup.
