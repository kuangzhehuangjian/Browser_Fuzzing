code
directive in module code

Description

Use this directive to auto-bootstrap an AngularJS application. The code directive designates the root element of the application and is typically placed near the root element of the page - e.g. on the code or code tags.

Only one AngularJS application can be auto-bootstrapped per HTML document. The first code found in the document will be used to define the root element to auto-bootstrap as an application. To run multiple applications in an HTML document you must manually bootstrap them using code instead. AngularJS applications cannot be nested within each other.

You can specify an AngularJS module to be used as the root module for the application. This module will be loaded into the code when the application is bootstrapped and should contain the application code needed or have dependencies on other modules that will contain the code. See code for more information.

In the example below if the code directive were not placed on the code element then the document would not be compiled, the code would not be instantiated and the code would not be resolved to code.

code is the easiest, and most common, way to bootstrap an application.
