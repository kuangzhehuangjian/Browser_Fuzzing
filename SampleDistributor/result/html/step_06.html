To dynamically generate links that will in the future lead to phone detail pages, we used the now-familiar double-curly brace binding in the code attribute values. In step 2, we added the code binding as the element content. In this step the code binding is used in the element attribute.

We also added phone images next to each record using an image tag with the code directive. That directive prevents the browser from treating the angular code markup literally, and initiating a request to invalid url code, which it would have done if we had only specified an attribute binding in a regular code attribute (code). Using the code directive prevents the browser from making an http request to an invalid location.
