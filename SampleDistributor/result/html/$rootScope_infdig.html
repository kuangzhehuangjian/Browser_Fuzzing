Description

This error occurs when the application's model becomes unstable and each code cycle triggers a state change and subsequent code cycle. Angular detects this situation and prevents an infinite loop from causing the browser to become unresponsive.

For example, the situation can occur by setting up a watch on a path and subsequently updating the same path when the value changes.

pre

The maximum number of allowed iterations of the code cycle is controlled via TTL setting which can be configured via code.
