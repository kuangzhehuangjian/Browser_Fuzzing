Description

A reference to the browser's code object. While code is globally available in JavaScript, it causes testability problems, because it is a global variable. In angular we always refer to it through the code service, so it may be overridden, removed or mocked for testing.

Expressions, like the one defined for the code directive in the example below, are evaluated with respect to the current scope. Therefore, there is no risk of inadvertently coding in a dependency on a global value in such an expression.
