code
directive in module code

Description

The following markup will make the button enabled on Chrome/Firefox but not on IE8 and older IEs: 

pre

The HTML specification does not require browsers to preserve the values of boolean attributes such as disabled. (Their presence means true and their absence means false.) If we put an Angular interpolation expression into such an attribute then the binding information would be lost when the browser removes the attribute. The code directive solves this problem for the code attribute. This complementary directive is not removed by the browser and so provides a permanent reliable place to store the binding information.
