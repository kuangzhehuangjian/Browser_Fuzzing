Description

Formats code to a string based on the requested code.

code string can be composed of the following elements:

  • code: 4 digit representation of year (e.g. AD 1 => 0001, AD 2010 => 2010)
  • code: 2 digit representation of year, padded (00-99). (e.g. AD 2001 => 01, AD 2010 => 10)
  • code: 1 digit representation of year, e.g. (AD 1 => 1, AD 199 => 199)
  • code: Month in year (January-December)
  • code: Month in year (Jan-Dec)
  • code: Month in year, padded (01-12)
  • code: Month in year (1-12)
  • code: Day in month, padded (01-31)
  • code: Day in month (1-31)
  • code: Day in Week,(Sunday-Saturday)
  • code: Day in Week, (Sun-Sat)
  • code: Hour in day, padded (00-23)
  • code: Hour in day (0-23)
  • code: Hour in am/pm, padded (01-12)
  • code: Hour in am/pm, (1-12)
  • code: Minute in hour, padded (00-59)
  • code: Minute in hour (0-59)
  • code: Second in minute, padded (00-59)
  • code: Second in minute (0-59)
  • code: Millisecond in second, padded (000-999)
  • code: am/pm marker
  • code: 4 digit (+sign) representation of the timezone offset (-1200-+1200)

code string can also be one of the following predefined localizable formats:

  • code: equivalent to code for en_US locale (e.g. Sep 3, 2010 12:05:08 pm)
  • code: equivalent to code for en_US locale (e.g. 9/3/10 12:05 pm)
  • code: equivalent to code for en_US locale (e.g. Friday, September 3, 2010)
  • code: equivalent to code for en_US locale (e.g. September 3, 2010)
  • code: equivalent to code for en_US locale (e.g. Sep 3, 2010)
  • code: equivalent to code for en_US locale (e.g. 9/3/10)
  • code: equivalent to code for en_US locale (e.g. 12:05:08 pm)
  • code: equivalent to code for en_US locale (e.g. 12:05 pm)

code string can contain literal values. These need to be quoted with single quotes (e.g. code). In order to output single quote, use two single quotes in a sequence (e.g. code).

dateDatenumberstring

Date to format either as Date object, milliseconds (string or number) or various ISO 8601 datetime string formats (e.g. yyyy-MM-ddTHH:mm:ss.SSSZ and its shorter versions like yyyy-MM-ddTHH:mmZ, yyyy-MM-dd or yyyyMMddTHHmmssZ). If no timezone is specified in the string input, the time is considered to be in the local timezone.

Formatting rules (see Description). If not specified, code is used.
