code
directive in module code

Description

The code directive removes or recreates a portion of the DOM tree based on an {expression}. If the expression assigned to code evaluates to a false value then the element is removed from the DOM, otherwise a clone of the element is reinserted into the DOM.

code differs from code and code in that code completely removes and recreates the element in the DOM rather than changing its visibility via the code css property. A common case when this difference is significant is when using css selectors that rely on an element's position within the DOM, such as the code or code pseudo-classes.

Note that when an element is removed using code its scope is destroyed and a new scope is created when the element is restored. The scope created within code inherits from its parent scope using prototypal inheritance. An important implication of this is if code is used within code to bind to a javascript primitive defined in the parent scope. In this case any modifications made to the variable within the child scope will override (hide) the value in the parent scope.

Also, code recreates elements using their compiled state. An example of this behavior is if an element's class attribute is directly modified after it's compiled, using something like jQuery's code method, and the element is later removed. When code recreates the element the added class will be lost because the original compiled state is used to regenerate the element.

Additionally, you can provide animations via the code module to animate the code and code effects.
