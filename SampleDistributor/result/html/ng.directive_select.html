directive in module code

Description

HTML code element with angular data-binding.

code

The code attribute can be used to dynamically generate a list of code elements for the code element using the array or object obtained by evaluating the code comprehension_expression.

When an item in the code menu is selected, the array element or object property represented by the selected option will be bound to the model identified by the code directive.

Note: code compares by reference, not value. This is important when binding to an array of objects. See an example in this jsfiddle.

Optionally, a single hard-coded code element, with the value set to an empty string, can be nested into the code element. This element will then represent the code or "not selected" option. See example below for demonstration.

Note: code provides an iterator facility for the code element which should be used instead of code when you want the code model to be bound to a non-string value. This is because an option element can only be bound to string values at present.

Usage

This directive can be used as custom element, but be aware of IE restrictions.

Where:

  • code / code: an expression which evaluates to an array / object to iterate over.
  • code: local variable which will refer to each item in the code or each property value of code during iteration.
  • code: local variable which will refer to a property name in code during iteration.
  • code: The result of this expression will be the label for code element. The code will most likely refer to the code variable (e.g. code).
  • code: The result of this expression will be bound to the model of the parent code element. If not specified, code expression will default to code.
  • code: The result of this expression will be used to group options using the code DOM element.
  • code: Used when working with an array of objects. The result of this expression will be used to identify the objects in the array. The code will most likely refer to the code variable (e.g. code).
