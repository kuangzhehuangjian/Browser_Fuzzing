  • when(path, route)

    Adds a new route definition to the code service.

    Parameters
    ParamTypeDetails
    pathstring

    Route path (matched against code). If code contains redundant trailing slash or is missing one, the route will still match and the code will be updated to add or drop the trailing slash to exactly match the route definition.

    • code can contain named groups starting with a colon: e.g. code. All characters up to the next slash are matched and stored in code under the given code when the route matches.
    • code can contain named groups starting with a colon and ending with a star: e.g.code. All characters are eagerly stored in code under the given code when the route matches.
    • code can contain optional named groups with a question mark: e.g.code.

    For example, routes like code will match code and extract:

    • code
    • code.
    routeObject

    Mapping information to be assigned to code on route match.

    Object properties:

    • code – code – Controller fn that should be associated with newly created scope or the name of a code if passed as a string.
    • code – code – A controller alias name. If present the controller will be published to scope under the code name.

    • code – code – html template as a string or a function that returns an html template as a string which should be used by ngView or code directives. This property takes precedence over code.

      If code is a function, it will be called with the following parameters:

      • code - route parameters extracted from the current code by applying the current route

    • code – code – path or function that returns a path to an html template that should be used by ngView.

      If code is a function, it will be called with the following parameters:

      • code - route parameters extracted from the current code by applying the current route

    • code - code - An optional map of dependencies which should be injected into the controller. If any of these dependencies are promises, the router will wait for them all to be resolved or one to be rejected before the controller is instantiated. If all the promises are resolved successfully, the values of the resolved promises are injected and $routeChangeSuccess event is fired. If any of the promises are rejected the $routeChangeError event is fired. The map object is:

      • code – code: a name of a dependency to be injected into the controller.
      • code - code: If code then it is an alias for a service. Otherwise if function, then it is code and the return value is treated as the dependency. If the result is a promise, it is resolved before its value is injected into the controller. Be aware that code will still refer to the previous route within these resolve functions. Use code to access the new route parameters, instead.

    • code – {(string|function())=} – value to update code path with and trigger route redirection.

      If code is a function, it will be called with the following parameters:

      • code - route parameters extracted from the current code by applying the current route templateUrl.
      • code - current code
      • code - current code

      The custom code function is expected to return a string which will be used to update code and code.

    • code - {boolean=} - reload route when only code or code changes.

      If the option is set to code and url in the browser changes, then code event is broadcasted on the root scope.

    • code - {boolean=} - match routes without being case sensitive

      If the option is set to code, then the particular route can be matched without being case sensitive
