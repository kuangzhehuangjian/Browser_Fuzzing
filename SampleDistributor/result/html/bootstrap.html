Automatic Initialization

Angular initializes automatically upon code event or when the code script is evaluated if at that time code is set to code. At this point Angular looks for the code directive which designates your application root. If the code directive is found then Angular will:

  • load the module associated with the directive.
  • create the application code
  • compile the DOM treating the code directive as the root of the compilation. This allows you to tell it to treat only a portion of the DOM as an Angular application.

Deferred Bootstrap

This feature enables tools like Batarang and test runners to hook into angular's bootstrap process and sneak in more modules into the DI registry which can replace or augment DI services for the purpose of instrumentation or mocking out heavy dependencies.

If code contains prefix code when code is called, the bootstrap process will be paused until code is called.

code takes an optional array of modules that should be added to the original list of modules that the app was about to be bootstrapped with.
