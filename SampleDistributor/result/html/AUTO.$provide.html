service in module code

Description

The code service has a number of methods for registering components with the code. Many of these functions are also exposed on code.

An Angular service is a singleton object created by a service factory. These service factories are functions which, in turn, are created by a service provider. The service providers are constructor functions. When instantiated they must contain a property called code, which holds the service factory function.

When you request a service, the code is responsible for finding the correct service provider, instantiating it and then calling its code service factory function to get the instance of the service.

Often services have no configuration options and there is no need to add methods to the service provider. The provider will be no more than a constructor function with a code property. For these cases the code service has additional helper methods to register services without specifying a provider.

  • code - registers a service provider with the code
  • code - registers a value/object that can be accessed by providers and services.
  • code - registers a value/object that can only be accessed by services, not providers.
  • code - registers a service factory function, code, that will be wrapped in a service provider object, whose code property will contain the given factory function.
  • code - registers a constructor function, code that that will be wrapped in a service provider object, whose code property will instantiate a new object using the given constructor function.

See the individual methods for more information and examples.

  • provider(name, provider)

    Register a provider function with the code. Provider functions are constructor functions, whose instances are responsible for "providing" a factory for a service.

    Service provider names start with the name of the service they provide followed by code. For example, the code service has a provider called code.

    Service provider objects can have additional methods which allow configuration of the provider and its service. Importantly, you can configure what kind of service is created by the code method, or how that service will act. For example, the code has a method code which lets you specify whether the code service will log debug messages to the console or not.

    Parameters
    ParamTypeDetails
    namestring

    The name of the instance. NOTE: the provider will be available under code key.

    providerObjectfunction()

    If the provider is:

    • code: then it should have a code method. The code method will be invoked using code when an instance needs to be created.
    • code: a new instance of the provider will be created using
      code, then treated as code.
