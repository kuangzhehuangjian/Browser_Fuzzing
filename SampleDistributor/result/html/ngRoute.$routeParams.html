Description

The code service allows you to retrieve the current set of route parameters.

Requires the code module to be installed.

The route parameters are a combination of code's code and code. The code parameters are extracted when the code path is matched.

In case of parameter name collision, code params take precedence over code params.

The service guarantees that the identity of the code object will remain unchanged (but its properties will likely change) even when a route change occurs.

Note that the code are only updated after a route change completes successfully. This means that you cannot rely on code being correct in route resolve functions. Instead you can use code to access the new route's parameters.
